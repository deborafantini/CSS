Organizing your CSS

Guia de Estilo definido

No seu projeto existe um guia de estilo já definido? Isso é muito importante para manter a consistência e facilitar que outras pessoas façam a leitura do seu documento. Caso você não possua (geralmente em projetos próprios), visite o guia da Mozilla para poder se orientar.

Mantenha a consistência

É muito importante ressaltar este ponto. A consistência beneficia outras pessoas que vão ler teu documento e facilita o seu trabalho, já que tu vai seguir um padrão. Lembrando que não existe apenas um forma certa de padronizar teus documentos, escolha a que goste mais e dê preferência em seguir sempre ela, para diminuir a carga mental.

Comentar não tira pedaço

É relevante gerar comentários entre as seções para deixar explícita onde ela começa e termina e comentar em momentos que você achar pertinente (seja para explicar uma propriedade que aprendeu recentemente, tutorial, ou o motivo de ter feito daquela maneira).

Crie uma sequência lógica no seu CSS

Comece de cima para baixo (body, h1, p, table...). Assim fica muito mais fácil para localizar o que deseja, principalmente quando se trata de projetos grandes.

Evite seletores muito específicos

Cada situação deve ser avaliada se um seletor convém ou não está ali. O mais importante é analisar se estar havendo uma duplicidade de informação, caso tenha, descarte este seletor e gere um que abranga todos ou a maioria.

Divida as folhas de estilo grandes em várias folhas menores

Em projetos onde há várias páginas, faz sentido ter mais de uma folha de estilo, sendo uma mais generalista e outra específica à página.

Metodologias CSS

Existem metodologias já predefinidas. Aqui irei apresentar as mais utilizadas:

OOCSS

É uma metodologia onde utiliza Orientação a Objetos no CSS, criando modelos padrões onde podem ser reutilizados na sua página web. Para maiores informações, entre em seu repositório.

BEM

BEM -> Block Element Modifier, neste método os elementos são vinculados a um bloco e existe uma nomenclatura de nomeação de classes e ids (geralmente com uso de travessão e underscore). Ela é muito utilizada atualmente e neste artigo você encontrará uma explicação ampla do seu conceito e porque usar.

Construindo Sistemas para o CSS

Uma outra maneira para organizar o seu CSS é usando uma abordagem mais programável, usando ferramentas de pré e pós-processador. Uma de pré-processador muito conhecida é o Sass e a de pós-processador é o cssnano.
